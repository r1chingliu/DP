# 技术方案

## 目标

项目需要满足三个要求：

1. 最终可在 iPhone 上使用
2. 支持持续迭代，便于通过 Git 在多台电脑接力开发
3. 先把核心体验做出来，再逐步接入 OCR、AI 和行情服务

## 技术栈

### 客户端

- `Expo`
- `React Native`
- `TypeScript`

### 服务端

后续推荐：

- `FastAPI`
- `PostgreSQL`
- `SQLAlchemy`
- `Redis`

服务端环境变量建议：

- `DASHSCOPE_BASE_URL`
- `DASHSCOPE_API_KEY`

不要把大模型密钥直接放进 React Native 客户端代码或提交到 Git。

## OCR 层

建议路线：

- 图片预处理
- OCR 提取
- 字段结构化与置信度评分
- 人工确认页

## AI 层

AI 只做：

- 阵型推荐
- 位置推荐
- 角色标签
- 阵容点评

AI 不做：

- 投资建议
- 买卖判断
- 收益预测

实现原则：

- 规则优先
- 模型润色
- 输出可落库，不在页面实时漂移

## 当前前端架构

### `src/types`

定义资产、球场位置、阵容状态等核心模型。

### `src/data`

当前先用模拟持仓数据验证界面和交互。

### `src/lib`

放排阵规则、位置映射、手动换位、锁位等纯逻辑。

### `src/components`

放首页卡片、球场、替补席、球星卡等 UI 组件。
